# 01 - 设计原则与边界

> 本文件定义游戏设计的核心原则、玩法边界（做什么/不做什么）与复杂度控制策略。
> 所有后续设计决策与功能取舍都应对照本文件。

---

## 1. 核心设计原则

### 1.1 双玩法缝合优先
- 本游戏的核心卖点 = **烽火撤离搜刮 × 六边形边建造攻防**。
- 任何设计决策必须服务于这两者的耦合，而非各自独立。
- 检验标准：移除任一玩法，另一玩法是否失去意义？若否，说明耦合不足。

### 1.2 风险与回报对等
- 烽火精髓：高价值物资 = 高撤离风险。
- 边防精髓：强建筑 = 高资源投入 + 被绕过/被破的风险。
- 任何"稳赚不赔"的机制都是设计缺陷，必须引入对应风险。

### 1.3 滚雪球抑制
- 领先者不应自动滚雪球到不可逆。
- 手段：落后者先手、装备丢失惩罚、劫镖翻盘、PvE 压力分散。
- 检验标准：第一名与最后一名的差距是否可在 2–3 轮内逆转？

### 1.4 决策密度优先于规则复杂度
- 每回合玩家应面对 3 个以上有意义的取舍。
- 取舍来源：构筑流派、建造位置、搜刮深度、撤离时机、劫掠/反劫掠。
- 避免为复杂而复杂的子系统。

### 1.5 多路径胜利
- 至少 4 条胜利路径（灰核归一/据点称霸/军火寡头/歼灭）。
- 不同流派玩家都应有可行获胜路线。

## 2. 玩法边界（做什么）

### 2.1 核心玩法（必须做）
- 六边形边建造系统
- 局前战备构筑（装备/负重/战备点）
- 禁区镇节点式搜刮
- 撤离路径劫掠
- 局外养成（军械库/据点/科技树）
- 六势力差异化
- 四幕剧情 + 全局事件
- 多胜利条件

### 2.2 辅助玩法（可做，优先级中）
- 士气系统
- 暂存点机制
- 保险箱机制
- 暗道/伏击隐蔽系统
- PvE 蚀体潮事件

### 2.3 扩展玩法（后续版本）
- 结盟/外交系统
- 雇佣兵/中立单位
- 自定义军械库配置保存
- 多地图/剧本

## 3. 玩法边界（不做什么）

### 3.1 明确排除
- **不做实时操作**：纯回合制，不引入 AP/MOBA 式实时元素。
- **不做 3D**：2D 平面表现，六边形网格。
- **不做内购付费赢**：养成进度仅来自局内表现。
- **不做无限地图**：固定尺寸地图，避免地图过大稀释冲突。
- **不做单位微操**：单位是"队"为单位，不控制单个士兵。
- **不做复杂经济模拟**：资源种类控制在 3–4 种。

### 3.2 暂不考虑
- 单人战役（MVP 聚焦多人）。
- 跨平台存档互通（取决于载体）。
- UGC/地图编辑器（后续扩展）。

## 4. 复杂度控制

### 4.1 复杂度预算
- 单局学习曲线目标：新手 2 局内理解全部核心机制。
- 单回合决策时间目标：≤ 3 分钟/人。
- 规则文档总量目标：核心规则 ≤ 30 页（桌游）/ 清晰分层（数字版）。

### 4.2 复杂度削减原则
- 能合并的子系统就合并（如士气系统 MVP 可省）。
- 能用通用机制就不用专属机制（如所有战斗共用一套结算公式）。
- 能查表就不算公式（数字版可自动结算，桌游用速查表）。

### 4.3 复杂度红线
- 同时存在的建筑类型 ≤ 12 种。
- 装备类型 ≤ 25 种。
- 禁区镇节点 ≤ 15 个。
- 单回合阶段数 = 6（不再增加）。

## 5. 体验目标

| 维度 | 目标 |
|---|---|
| 紧张感 | 撤离时"能不能带出去"的心跳时刻 |
| 博弈感 | 边防选址与劫镖预判的智力对抗 |
| 成长感 | 军械库从手枪到狙击枪的积累 |
| 叙事感 | 四幕剧情推进的史诗感 |
| 反转感 | 落后者劫镖翻盘的爽快 |

## 6. 设计决策仲裁原则

当多个设计方案冲突时，按以下优先级裁决：
1. 双玩法耦合 > 单玩法深度
2. 风险回报平衡 > 数值华丽
3. 滚雪球抑制 > 强者体验
4. 决策密度 > 规则丰富度
5. 多路径胜利 > 主线唯一性
