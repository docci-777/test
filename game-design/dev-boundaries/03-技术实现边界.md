# 03 - 技术实现边界

> 本文件定义游戏载体假设、技术实现边界、性能/复杂度约束与可扩展性要求。
> 用于指导后续技术选型与开发规划。

---

## 1. 载体假设

### 1.1 主载体（建议）
**数字版（PC/移动端）**
- 理由：六边形边建造 + 节点式禁区 + 多单位结算，数字版可自动处理大量计算与状态。
- 烽火式装备库存管理在数字版体验远优于桌游。
- 联机 6 人回合制，数字版天然支持。

### 1.2 备选载体
**实体桌游**
- 适合 2–4 人小规模体验。
- 需大量 token/卡牌/骰子，制作成本高。
- 装备库存需用卡牌管理，复杂度上限低。
- 可作为数字版的"精简体验版"。

### 1.3 本文件默认
后续技术边界以**数字版**为准。桌游版需另行裁剪。

## 2. 核心系统模块划分

| 模块 | 职责 | 依赖 |
|---|---|---|
| 地图系统 | 六边形网格、地形、边建筑状态 | — |
| 回合系统 | 六阶段流程、玩家顺序、行动点 | 地图、单位 |
| 单位系统 | 搜刮队/边防军/工兵属性与移动 | 地图 |
| 构筑系统 | 局前装备配置、负重/战备点 | 单位、军械库 |
| 禁区系统 | 节点地图、搜刮、AI 蚀体 | 单位、构筑 |
| 战斗系统 | 结算公式、先手、掉落 | 单位、地图 |
| 撤离系统 | 路径规划、劫掠触发 | 地图、战斗 |
| 养成系统 | 军械库、据点、科技树 | — |
| 势力系统 | 天赋/技能 | 单位、战斗 |
| 剧情系统 | 四幕事件、胜利判定 | 全部 |
| 联机系统 | 6 人同步、状态广播 | 全部 |

## 3. 状态管理边界

### 3.1 局外状态（持久化）
- 军械库等级与库存
- 据点等级
- 科技树进度
- 玩家历史战绩

### 3.2 局内状态（单局）
- 地图与边建筑
- 所有单位位置与属性
- 禁区镇节点状态
- 资源持有量
- 当前回合/阶段/行动玩家
- 剧情进度

### 3.3 状态序列化要求
- 局内状态必须可完整序列化（支持存档/断线重连）。
- 每次状态变更需可回放（支持观战/复盘/反作弊）。
- 状态变更采用**事件驱动**（command 模式），便于撤销与同步。

## 4. 性能与复杂度边界

### 4.1 单局状态规模上限
| 项 | 上限 | 说明 |
|---|---|---|
| 地图六边形数 | ≤ 200 | 半径 6 约 127 格 |
| 边建筑总数 | ≤ 300 | 每边 1 建筑 |
| 单位总数（全场） | ≤ 60 | 6 人 × 10 |
| 禁区镇节点数（单镇） | ≤ 15 | |
| 禁区镇数 | 6 | 固定 |
| 装备种类 | ≤ 25 | |
| 建筑种类 | ≤ 12 | |

### 4.2 单回合计算量
- 战斗结算：O(单位数 × 战斗次数)，可忽略。
- 路径规划（撤离）：六边形 A*，地图小，可忽略。
- AI 蚀体决策：规则驱动，无需复杂 AI，O(蚀体数)。
- **结论**：无性能瓶颈，普通设备可流畅运行。

### 4.3 联机同步
- 回合制 = 低频状态同步，对网络要求低。
- 关键：断线重连、状态校验、防作弊。
- 建议：权威服务器 + 客户端预测（回合制下预测简单）。

## 5. 随机性实现

| 随机源 | 实现 | 公平性 |
|---|---|---|
| 战斗骰子 | 服务器端 RNG | 可审计 |
| 物资分布 | 开局种子生成 | 全场一致 |
| 蚀体行为 | 规则 + 加权随机 | 可复现 |
| 事件触发 | 按轮次固定 + 随机修饰 | 主线可控 |

- 所有随机数由服务器生成并广播，客户端不可本地决定战斗结果。
- 支持种子回放（赛事/复盘用）。

## 6. 可扩展性边界

### 6.1 数据驱动
- 装备、建筑、势力、事件、节点类型均**数据配置化**（JSON/表）。
- 数值平衡调整不需改代码，改配置即可。
- 便于后续新增内容（DLC/赛季）。

### 6.2 模块解耦
- 各系统模块通过事件/接口通信，避免硬耦合。
- 禁区系统可独立测试（节点图逻辑与主地图分离）。
- 战斗系统可独立测试（纯数值结算）。

### 6.3 预留扩展点
- 势力数（当前 6，预留扩展接口）
- 地图剧本（当前 1，预留多剧本加载）
- 胜利条件（当前 4，预留配置化）
- 装备/建筑（数据驱动，天然可扩展）

## 7. 技术红线

| 红线 | 说明 |
|---|---|
| 不引入实时战斗 | 严格回合制 |
| 不做 3D 渲染 | 2D 平面 |
| 不做物理引擎 | 六边形离散网格，无需物理 |
| 不依赖外部服务（除联机） | 单机可玩（人机） |
| 不做复杂 AI | 蚀体规则驱动，人机可选但非重点 |
